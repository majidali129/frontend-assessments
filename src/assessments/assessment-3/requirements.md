# Frontend Practice Assessment 3

## React + JavaScript

## Project: Product Explorer App

### Objective

Build a _Product Explorer App_ that fetches product data from a public API and allows users to search, filter, sort, and view details.

---

## Requirements

### Core Features

- Fetch products from a public API such as:
  https://fakestoreapi.com/products
- Display product cards with:
  - Image
  - Title
  - Price
  - Category

- Add _search_ by product title
- Add _filter by category_
- Add _sort options_:
  - Price: Low to High
  - Price: High to Low
  - Title: A–Z

- On clicking a product:
  - Show product details in a side panel or modal:
    - Full title
    - Description
    - Category
    - Price
    - Rating

---

## Functional Expectations

- Use _functional components_
- Use _React hooks_
- Handle:
  - loading
  - error
  - empty state

- Keep logic modular and reusable
- Maintain selected filters and sort state correctly

---

## UI Requirements

- Responsive layout
- Clean product grid
- Clear active filters
- Highlight selected product
- Good empty state messaging

---

## Bonus

- Debounced search
- Pagination
- Persist filters in query params
- Skeleton loader
- Custom hooks

---

## Evaluation Criteria

- API handling
- State organization
- Filter/sort/search logic
- Component structure
- Responsive UI
- Clean code
